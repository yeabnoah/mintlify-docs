# Global Styles & Theming (`globals.css`)

Welcome to the global styling and theming guide for the Ultimate Cursor AI Boilerplate. This document provides an overview of how our `globals.css` file is structured, what styles it defines, and how it powers the look and feel of the entire application.

## Purpose of `globals.css`

The `globals.css` file is the single source of truth for global styles in our Next.js project. It:

- Establishes a consistent design foundation across all pages and components
- Defines CSS custom properties (variables) for colors, spacing, fonts, and more
- Implements theming (including dark mode)
- Integrates Tailwind CSS and utility layers
- Adds custom keyframes and utility classes for advanced UI effects

## File Structure & Key Sections

### 1. Imports

```css
@import "tailwindcss";
@import "tw-animate-css";
```

We import Tailwind CSS and any additional utility layers (like `tw-animate-css`) at the top, ensuring all Tailwind utilities and custom animations are available globally.

### 2. CSS Custom Properties (Variables)

We use modern CSS custom properties to define our color palette, radii, shadows, and font stacks. This enables easy theming and consistent usage throughout the app.

```css
:root {
  --background: oklch(1.0000 0 0);
  --foreground: oklch(0.2686 0 0);
  --primary: oklch(0.7686 0.1647 70.0804);
  --radius: 0.375rem;
  --font-sans: Inter, sans-serif;
  /* ...more variables for cards, borders, charts, etc... */
}
```

#### Dark Mode Variables

We override the root variables inside a `.dark` selector to provide a dark theme:

```css
.dark {
  --background: oklch(0.2046 0 0);
  --foreground: oklch(0.9219 0 0);
  --primary: oklch(0.7686 0.1647 70.0804);
  /* ...other dark mode overrides... */
}
```

### 3. Theming Layer

We use an `@theme inline` block to map our custom properties to semantic names, making it easier to reference them in components and utility classes:

```css
@theme inline {
  --color-background: var(--background);
  --color-primary: var(--primary);
  --font-sans: var(--font-sans);
  /* ...other mappings... */
}
```

### 4. Tailwind Base Layer & Resets

We apply Tailwind's base layer and extend it with our own resets and font settings:

```css
@layer base {
  * {
    @apply border-border;
  }
  body {
    @apply bg-background text-foreground;
    font-family: var(--font-sans);
    letter-spacing: var(--tracking-normal);
  }
}
```

This ensures all elements use our border color and the body uses the correct background, text color, and font.

### 5. Custom Keyframes & Utility Classes

We define custom keyframes for advanced UI effects, such as marquees:

```css
@keyframes marquee {
  from { transform: translateX(0); }
  to { transform: translateX(calc(-100% - var(--gap))); }
}

.marquee {
  animation: marquee var(--duration) linear infinite;
}
```

Similar keyframes and classes exist for vertical marquees and other effects.

## How Styles Are Applied Globally

- **Tailwind Utilities:** All Tailwind classes are available globally, and our custom variables are referenced in the Tailwind config for seamless integration.
- **CSS Variables:** Components and pages use CSS variables (e.g., `bg-background`, `text-foreground`) for consistent theming.
- **Dark Mode:** The `.dark` class on the `<html>` or `<body>` element toggles dark mode, instantly switching the theme by overriding variables.
- **Custom Utilities:** Classes like `.marquee` can be used anywhere in the app for advanced animations.

## Theming Approach & Customizations

### Modern CSS Variables
- We use [OKLCH color space](https://oklch.com/) for perceptually-uniform colors, making it easier to manage accessible and visually pleasing themes.
- All theme values (colors, radii, shadows, fonts) are defined as variables, making global changes trivial.

### Tailwind Integration
- Tailwind's utility classes are mapped to our custom properties, so you can use classes like `bg-background` or `text-primary` and they will always reflect the current theme.
- The Tailwind config is extended to reference these variables, ensuring design tokens are consistent everywhere.

### Dark Mode
- Dark mode is handled by toggling the `.dark` class, which swaps out the relevant variables for their dark equivalents.
- No need for separate dark/light classes on every componentâ€”just toggle `.dark` at the root.

### Custom Tweaks
- We use the `@theme inline` block for semantic variable mapping, which is not standard in all setups.
- Custom keyframes and utility classes (like `.marquee`) are defined for advanced UI needs.
- Some resets and base styles are layered on top of Tailwind's defaults for extra consistency.

## Example Usage in a Component

```tsx
<div className="bg-background text-foreground p-6 rounded-lg shadow">
  Welcome to the Ultimate Cursor AI Boilerplate!
</div>
```

This component will automatically adapt to the current theme and global styles.

## Assumptions & Notes

- We assume all theme tokens are referenced in the Tailwind config for utility class support.
- The use of OKLCH and semantic variable mapping is a modern, accessibility-focused approach.
- If you need to add or tweak a theme value, update it in `globals.css` and (if needed) the Tailwind config.
